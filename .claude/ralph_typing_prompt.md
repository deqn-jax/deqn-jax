# Pattern precedent

Commit `3ae741f` ("typing: configure Pyright + fix two annotation lies") cleared ~240 of 362 errors via two structural fixes — neither was a "real bug", both were *annotation lies* (declared types that didn't match what the runtime stored/returned):

- `Metrics(loss: float, residuals: Dict[str, float], grad_norm: float)` was wrong — every constructor stores JAX `Array`s. Fix: change the declared types.
- `VariableSpec.unpack_state/unpack_policy` returned `NamedTuple` (abstract base) but the runtime tuple has dynamic fields built from `state_names`. Fix: change return type to `Any` and document why.

**Look for analogous lies first.** They produce the largest cleanups per LOC changed.

Other categories (in decreasing leverage):

- **REAL_BUG**: fix required (e.g., `benchmark.py:67` calls `train_step()` without `lr_scale`; `benchmark.py:68` accesses `.params` on the 3-tuple from `create_train_state`). Cheap fixes.
- **JAX_NOISE / EQX_NOISE**: framework limitations Pyright can't see through (Equinox `Module.__call__`, JAX tracer types). Suppress with `# pyright: ignore[code]  # one-line reason`.
- **PYDANTIC_DICT**: `OptimizerConfig(**opt_dict)` style — Pyright can't validate `**dict[str, Any]` against typed `__init__`. Suppress per-line with reason. Don't rewrite YAML loading.
- **OPTIONAL_NARROWING**: `model.step_fn(...)` where `step_fn: Optional[Callable]`. Sometimes adding `assert model.step_fn is not None` (with reason) is correct; sometimes the field shouldn't be Optional at all.

# Phase 1 protocol — audit

Run when `docs/typing_audit.md` does not exist.

1. `uvx ty check src/ > /tmp/typing.out 2>&1` — capture full output.
2. Build per-file error counts: `rg '^\s+/.*\.py:\d+:\d+ - error:' /tmp/typing.out | rg -oP 'deqn_jax/[^:]+\.py' | sort | uniq -c | sort -rn`.
3. For each file with ≥ 3 errors, sample the actual messages and classify into these buckets. For files with < 3 errors, batch them into "MISC" with file:line refs.
4. Write `docs/typing_audit.md` with this exact structure:

   ```markdown
   # Typing Audit (deqn-jax src/)

   Generated by Ralph Loop typing iteration. ty version + date in header.

   ## Summary
   - Total: <N> diagnostics
   - Bucket counts: REAL_BUG=<n>, ANNOTATION_LIE=<n>, JAX_NOISE=<n>, PYDANTIC_DICT=<n>, OPTIONAL_NARROWING=<n>, DECISION_NEEDED=<n>
   - Stop target: ≤ 30 diagnostics

   ## Workplan (ordered by leverage)

   ### 1. <FILE_OR_CATEGORY>  [BUCKET]  [STATUS: TODO]
   <error sample(s) with file:line>
   **Plan:** <one sentence on what to do>
   **Cost:** <S/M/L>

   ### 2. ...
   ```

   Status field is `TODO` initially; phase-2 iterations flip it to `DONE` or `SUPPRESSED` (or `BLOCKED` with reason if you have to bail).

5. `git add docs/typing_audit.md && git commit -m "typing: triage audit of src/ (Ralph Loop phase 1)"` and exit.

# Phase 2 protocol — one item per iteration

Run when `docs/typing_audit.md` exists.

1. Find the first item in the audit with `[STATUS: TODO]`. Read its bucket + plan.
2. State your one-sentence plan.
3. Make the smallest change consistent with the bucket:
   - **REAL_BUG / ANNOTATION_LIE**: fix the code/types.
   - **JAX_NOISE / EQX_NOISE / PYDANTIC_DICT**: add `# pyright: ignore[<code>]  # <reason>` at the offending line(s). Reason must be specific (not "noise").
   - **OPTIONAL_NARROWING**: if the field is genuinely required, change it to non-Optional in `types.py`. If it's truly optional, add `assert <field> is not None, "<reason>"` before the call.
4. Run `uv run pytest tests/ -q`. Pass count must be ≥ 309. (1 skipped is normal.)
5. Run `uvx ty check src/ 2>&1 | tail -1` and confirm the count went down (or, for SUPPRESSED items, the specific error type went down).
6. Mark the audit item `[STATUS: DONE]` (or `[STATUS: SUPPRESSED]` if you used `# pyright: ignore`). Update the summary line counts.
7. If tests fail: `git restore .`, change the audit item to `[STATUS: BLOCKED: <one-line reason>]`, commit ONLY the audit edit, and exit so a human can review.
8. If tests pass: `git add -A && git commit -m "typing: <focused message referencing audit section>"` and exit.
