# What "decomposition" means

Mirror the existing `_eq2_diagnostics` / `_eq4_diagnostics` template in `src/deqn_jax/models/disaster/diagnostics.py`:

- Identify all summands of the residual (LHS, RHS, expectation term).
- Log mean for each summand so relative magnitudes are visible (which term dominates? is the expectation term tiny vs the lhs?).
- For any RATIO that drives an exponent (e.g. `pi_tilda/pi`, `pi_w_tilda*mu_z_ss/pi_w`, `i_n/i`), log mean/std/min/max — these are the things that go nonlinear and blow up.
- Log a `log_residual` (`log(LHS) - log(RHS)`) mean/std as an order-of-magnitude consistency check.
- For any soft-floored quantity (`K_p_inner`, `K_w_inner`, `habit_now`, `habit_next`), log a saturation fraction (`mean(x < floor + eps)`).
- Log relevant exponents (constants) for sanity, namespaced under the equation.

Output keys must be namespaced `eq{N}_diag/<name>` so they don't collide.
